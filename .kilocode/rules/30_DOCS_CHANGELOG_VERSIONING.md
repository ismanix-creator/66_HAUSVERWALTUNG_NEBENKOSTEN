# 30_DOCS_CHANGELOG_VERSIONING – Dokumentation & Versionierung

16. Jede fachliche Änderung erfordert eine entsprechende Dokumentationspflege.
17. Der Abschluss eines Schrittes setzt ein Update von TODO- oder Statusdateien voraus.
18. Änderungen an der Konfiguration erfordern ein Update der Referenzdokumentation.
19. Relevante Änderungen erhalten verpflichtende Changelog-Einträge.
20. Dokumentation und tatsächliches Verhalten dürfen sich nicht widersprechen.
